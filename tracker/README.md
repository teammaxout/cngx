# Cogscope Drift Tracker

A **free, static** public view of behavioral drift over time — built from opt-in community submissions, not a maintainer-funded scraper.

- **Cost to run:** $0 (GitHub Pages + Actions on a public repo)
- **Data source:** `cogscope submit` (anonymous numeric fingerprints only)
- **No prompts, no outputs, no API keys** in the dataset

## Directory layout

```
tracker/

├── build.py            # Reads data/, writes site/
├── data/
```

## Record schema (version 1)

Each submission is a single JSON object. Community files live at `tracker/data/community/<record_id>.json`.

| Field | Type | Description |
|-------|------|-------------|
| `schema_version` | int | Always `1` for this format |
| `record_id` | string | UUID for this submission |
| `timestamp` | string | ISO-8601 UTC time of the captured fingerprint |
| `model` | string | Model name (e.g. `gpt-4o-mini`) |
| `baseline_label` | string | Submitter's pinned baseline name (opaque label) |
| `drift_score` | float | 0–1 drift vs that baseline |
| `depth` | int | Reasoning depth |
| `verification_steps` | int | Self-verification step count |
| `hedging_ratio` | float | Uncertainty / confidence ratio |
| `branching_factor` | float | Branching in reasoning |
| `total_steps` | int | Total reasoning steps |
| `correction_count` | int | Self-correction markers |
| `uncertainty_markers` | int | Hedging phrase count |
| `output_length` | int | Output character count |
| `reasoning_length` | int | Reasoning character count |

**Optional (samples only):**

| Field | Type | Description |
|-------|------|-------------|
| `sample` | bool | `true` = demonstration data, not a live measurement |
| `_comment` | string | Human note in sample files only — stripped from user submit |

**Never included:** `prompt`, `output`, `reasoning`, `trace_id`, `task_id`, or any free-text content.

## Submitting data

From a machine with local Cogscope captures and a pinned baseline:

```bash
cogscope pin --label my-baseline
cogscope watch   # or capture traffic locally
cogscope submit --baseline my-baseline --dry-run   # preview exact JSON
cogscope submit --baseline my-baseline             # confirm, then PR or pending/
```

If `gh` is authenticated, `submit` opens a PR adding one file under `tracker/data/community/`. Otherwise it writes to `tracker/data/community/pending/` with manual PR instructions.

## Building the site locally

```bash
python tracker/build.py
# Open tracker/site/index.html in a browser (or serve with: python -m http.server --directory tracker/site)
```

The build reads all `*.json` under `data/samples/`, `data/community/`, and `data/community/pending/` (skips `annotations.json`).

## Deployment

`.github/workflows/tracker-deploy.yml` runs `build.py` on push to `main` (and daily), then deploys `tracker/site/` to GitHub Pages. No paid services.

## Model-update annotations

`data/annotations.json` holds vertical-line markers on charts. **Only add entries with a verifiable source URL.** Placeholder `_todo` entries are ignored by the build until a human adds a real `date` and `source_url`.

## Optional maintainer probe (OFF by default)

`.github/workflows/tracker-probe.yml` can run a small fixed battery of prompts against configured models **only when** the repository variable `ENABLE_TRACKER_PROBE` is set to `true`.

- **Default:** disabled — tracker works on community data alone
- **Cost:** real money if enabled (API keys via repository secrets)
- **Budget cap:** hard-coded at **5 API calls maximum per run** in the workflow

Do not enable unless you accept ongoing API spend. The probe is a convenience for maintainers, not a requirement.

## Sample data

Files in `data/samples/` are explicitly labeled `"sample": true`. They exist to prove the build pipeline with zero API spend. They are not presented as live benchmark results.
